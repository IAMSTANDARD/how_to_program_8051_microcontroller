Build the development environment of AT89S52, S51 + SDCC + USBASP under Ubuntu
51 of MCU development under Linux

-51 series of development environment to build

Contact: sunxiao.gin@gmail.com

Attachment package content:  program source code, super simple Makefile template, usbasp can burn 51 firmware programs, related documents.

http://download.csdn.net/detail/galaxy_blue/4327653

Series Description: Program development under LINUX has certain maturity, including most MCU, FPGA, DSP and even PCB drawing are feasible. This series will target 51 series MCUs.

The environment of this article is as follows:

OS system: ubuntu 12.04

Compiler: sdcc

Burning software: avrdude

Burning device: usbasp

Development board:

There is indeed a small development board, but this time I deliberately joined a personal version of the small breadboard. For this reason, do n’t be afraid to develop MCUs without hardware foundation. Just do it. In fact, development is so easy.

As shown below

It is best to get only 1.2V * 4 battery when running. It is feasible to use three 3.6V (I forgot to bring my battery holder, connect it directly) ~ It is recommended to have a 5V power supply connected by USB or yourself There is a regulated power supply made of a transformer, and a small power board on the breadboard is such a function, but it is not used here.

Programming: Vim (upgraded version of Notepad, very easy to use, highly recommended)


ready:

For software installation, it is recommended to use ubuntu's software center for software installation, which is more convenient. Need sdcc , avrdude is enough. Text editing will do. Can be integrated in codeblocks and eclipse (51 has not done it), there is a 51MCU IDE under ubuntu, you can use 'mcu8051ide' down.

For the command line can be installed as follows

[plain]  view plain copy
  1. sudo apt-get install vim  
  2. sudo apt-get install avrdude   

Then they are all installed. The next step can begin.


Source program:

Before you start, you need to write a main.c program

This is a very simple program to make a blink.

[cpp]  view plain copy
  1. #include <8052.h>  
  2. typedef unsigned int size_t;  
  3.    
  4. #define LED P0_0  
  5.    
  6. void delay(size_t t)  
  7. {  
  8.         while(t--);  
  9. }  
  10.    
  11. void main()  
  12. {  
  13.         while(1)  
  14.         {  
  15.                 LED = 0;  
  16.                 delay(20000);  
  17.                 LED = 1;  
  18.                 delay(20000);  
  19.         }  
  20. }  


Compile:

After that, we must compile the program to turn the program into * .elf * .bin * .hex and the like that can be used by the single chip microcomputer

Here we are using * .ihx (intel hex) which appears in SDCC default compilation

Execute in current directory

[plain]  view plain copy
  1. sdcc -mmcs51 main.c  
Note: No -mmcs51 is possible.

A lot of files will be generated under the current directory ~ remember the main.ihx we need ~


Configure avrdude:

Write with avrdude 

We can't directly use avrdude to program, because it only supports AVR chip by default, but with some configuration, we can use it to program.

For the 8051 chip, we have three configuration files, which are suitable for different models. See which one you want, then copy to /etc/avrdude.conf and save.

(Sudo gedit /etc/avrdude.conf)

Get it done ~

Option description (Options):

[plain]  view plain copy
  1. -p specifies the type of the MCU connected to the programmer.  
  2. -c specifies the default programmer  
  3. -e causes a chip erase to be executed.  
  4. -U memtype:op:filename  
  5. The op field specifies what operation to perform:  
  6.  r read device memory and write to the specified file  
  7.  w read data from the specified file and write to the device memory  
  8.  v read data from both the device and the specified file and perform a verify
